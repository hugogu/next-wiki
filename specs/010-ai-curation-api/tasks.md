## Dependencies & Execution Order

### Phase Dependencies

- **Setup (Phase 1)**: No dependencies — baseline verification.
- **Foundational (Phase 2)**: Depends on Setup. **BLOCKS** all user stories. T011 depends on T004-T010 (permission wiring). T012-T014 depend on T011 (the `ai.read` scope must exist before the new tests can be written meaningfully). T015-T016 can proceed after Setup and do not depend on the retrieval filter; both must complete before US1/US2/US4.
- **User Stories (Phase 3-7)**: Each depends on Foundational completion. US1/US2 (frontmatter), US3 (semantic), US4 (links), US5 (batch) are independent of each other.
- **MCP (Phase 8)**: Depends on US1-US5 completion (the v1 endpoints must exist before MCP tools can wrap them).
- **Polish (Phase 9)**: Depends on MCP completion (the verification suite covers MCP + v1 + tests together).

  ↓

  └── T015-T016 (frontmatter parser) ─┘
  ↓
US1 (6 tasks) ∥ US2 (9 tasks) ∥ US3 (10 tasks) ∥ US4 (12 tasks) ∥ US5 (9 tasks)

  ↓
Phase 9 Polish (6 tasks)
```

- **Test tasks [P]**: tests for distinct code paths can run in parallel.

```
